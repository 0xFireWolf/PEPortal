导入应用程序（进阶）

导入应用程序（进阶）

本小节将详细介绍导入应用程序的进阶方法。

新版启动器支持直接从文件快速导入应用程序。我们暂且称这个文件为『定义文件』

方法一：

单击菜单上的『从文件导入应用程序』，在弹出的对话框中选择一个或多个『定义文件』后即可完成导入。

方法二：

单击菜单上的『管理应用程序』，直接将一个或多个『定义文件』拖入到弹出的窗口中即可。

定义文件介绍

定义文件是一个属性表（plist）文件，里面定义了应用程序的路径、说明以及所属的分类。

启动器支持两种定义文件格式，一种是字典格式，一种是纯数组格式。

纯数组格式

纯数组格式简单易懂，也是应用管理器默认的导出格式。

我们先来看看对应的属性表文件的内容是什么样子的。

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<array>
    <dict>
        <key>BundlePath</key>
        <string>/System/Library/CoreServices/Finder.app</string>
        <key>Category</key>
        <string>FileManagement</string>
        <key>LocalizedDescriptions</key>
        <dict>
            <key>en-US</key>
            <string>Apple OS X's File Manager</string>
            <key>zh-Hans</key>
            <string></string>
            <key>zh-Hant</key>
            <string></string>
        </dict>
    </dict>
    <dict>
        <key>BundlePath</key>
        <string>/Applications/BetterZip.app</string>
        <key>Category</key>
        <string>FileManagement</string>
        <key>LocalizedDescriptions</key>
        <dict>
            <key>en-US</key>
            <string>A famous OS X archive manager</string>
            <key>zh-Hans</key>
            <string></string>
            <key>zh-Hant</key>
            <string></string>
        </dict>
    </dict>
</array>
</plist>

我们推荐你使用 Xcode，PlistEdit Pro 等工具来编辑属性表文件。更直观也更安全。

从上述文本中我们可以看到，纯数组格式的属性表文件就是一个数组 <array>...</array>，数组里的每一个元素都是一个字典。

这个字典里包含了如下三个键值：

  • BundlePath：它对应的值的类型为字符串，定义了应用程序包的路径。（注意：不是二进制的路径）
  • Category：它对应的值的类型也为字符串，定义了应用程序的分类。可选分类为如下：
    1. FileManagement: 对应『文件管理』分类
    2. DiskManagement: 对应『磁盘管理』分类
    3. HardwareManagement: 对应『硬件管理』分类
    4. DocumentEditor: 对应『文档编辑』分类
    5. BootloaderUtility: 对应『引导工具』分类
    6. MaintenanceKits: 对应『维护套件』分类
    7. Unspecified: 对应『未分类』分类
  • LocalizedDescriptions：它对应的值的类型为字典，定义多国语言的应用介绍。目前只支持三种语言：英语（en-US）简体中文（zh-Hans）以及繁体中文（zh-Hant）。每种语言对应的值都为字符串。

字典格式

在这种格式下，属性表的键（Key）均为应用程序分类，对应的值（Value）是一个数组，包含了隶属于这个分类下的应用程序信息。

我们先来看一看这种格式下的文档内容是什么样子的。

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>BootloaderUtility</key>
    <array>
        <dict>
            <key>BundlePath</key>
            <string>/Applications/Chameleon Wizard.app</string>
            <key>LocalizedDescriptions</key>
            <dict>
                <key>en-US</key>
                <string>Edit Chameleon's boot configurations</string>
                <key>zh-Hans</key>
                <string></string>
                <key>zh-Hant</key>
                <string></string>
            </dict>
        </dict>
    </array>
    <key>DiskManagement</key>
    <array>
        <dict>
            <key>BundlePath</key>
            <string>/Applications/Utilities/Disk Utility.app</string>
            <key>LocalizedDescriptions</key>
            <dict>
                <key>en-US</key>
                <string>Apple's latest Disk Utility</string>
                <key>zh-Hans</key>
                <string></string>
                <key>zh-Hant</key>
                <string></string>
            </dict>
        </dict>
    </array>
</dict>
</plist>

我们可以看到『引导工具』分类下是一个数组，数组的每个元素是一个字典。

除了没有 Category 这个键以外，这个字典的格式与之前的数组格式是一样的。

请注意：请勿重新定义 Category 键，因为在这种格式下 Category 的值会被忽略。
